# Contributing

Daisy is **source-visible, not community-run**. The source is published so you
can read it and verify what the app does — in particular, that your audio and
transcripts never leave your machine.

## Issues: yes

Bug reports and feature requests are welcome. Please include your platform,
Daisy version, and build SHA (Settings → About). If a bug involves a specific
recording, describe the setup — never attach meeting audio or transcripts.

## Pull requests: no

We don't accept code contributions, and PRs are closed without review. This
keeps the copyright singular and the maintenance load sustainable for a small
team. If you've found a bug, an issue with a clear description is genuinely
more useful to us than a patch.

## Security issues

Please don't open public issues for security problems — see
[SECURITY.md](SECURITY.md).

## Building

See [docs/BUILDING.md](docs/BUILDING.md). You're welcome to build and run
Daisy for your own personal use per the license. The paid product is the
signed, notarized, auto-updating build — plus it's what funds development.
